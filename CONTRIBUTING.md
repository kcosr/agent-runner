# Contributing to task-runner

Thanks for your interest. task-runner is a Node CLI + TypeScript
monorepo; this short note covers the practical bits.

## Getting set up

```bash
git clone https://github.com/kcosr/task-runner.git
cd task-runner
npm install
npm run build
npm link -w task-runner    # puts `task-runner` on your PATH
```

The `-w task-runner` flag targets the `apps/cli/` workspace where the
binary lives; a bare `npm link` from the repo root would link the
private workspace root instead.

Then copy the bundled templates into your config dir if you want to
use them by name — see "Install the bundled templates" in the
[README](README.md#install-the-bundled-templates).

## Repo layout

- `apps/cli/` — CLI entry point and daemon transport
- `apps/web/` — bundled browser dashboard
- `packages/core/` — run loop, manifest, backend adapters, assignment
  parsing, hooks, command service
- `agents/`, `assignments/`, `tasks/` — bundled definitions
- `docs/` — user-facing documentation
- `scripts/` — release and migration scripts

Primary entry points:

- `apps/cli/src/cli.ts`
- `packages/core/src/core/run/run-loop.ts`
- `packages/core/src/core/run/manifest.ts`
- `packages/core/src/core/commands/service.ts`

See [AGENTS.md](AGENTS.md) for agent-facing onboarding notes, which
double as a quick orientation for human contributors.

## Coding conventions

- TypeScript, ESM modules.
- Source of truth lives under `apps/cli/src/` and `packages/core/src/`;
  workspace `dist/` output is generated by `npm run build` and not
  committed.
- Formatting and linting are handled by Biome (`biome.json`);
  pre-commit hooks run `lint-staged`.
- Keep edits ASCII-only unless the file already uses Unicode.
- Prefer end-state implementations over transitional or
  fallback-laden code. See [CLAUDE.md](CLAUDE.md) for fuller
  guidance.

## Before committing

Run the standard verification pipeline:

```bash
npm run build
npm run lint
npm run format:check
npm run imports:check
npm run test
```

Or run them as one step:

```bash
npm run check
```

For dependency / unused-export changes:

```bash
npm run check:knip
```

To apply autofixes:

```bash
npm run lint:fix
npm run format
npm run imports:fix
```

## Pull requests

- Keep changes focused — one feature or fix per PR.
- Add a brief entry under `## [Unreleased]` in
  [CHANGELOG.md](CHANGELOG.md) for user-visible changes.
- Update relevant docs alongside code changes — README, `docs/`,
  inline comments where they earn their keep.
- Reference scope when proposing a feature: see
  [docs/scope.md](docs/scope.md) for the triage heuristic.

## Reporting bugs / requesting features

Open an issue on GitHub with a minimal reproduction (a passive run is
usually easiest) and the output of `task-runner status`.
